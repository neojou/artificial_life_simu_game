## 1. 高層概念 (High Concept)

**類型**：自主生命模擬遊戲（Autonomous Agent Simulation）、療癒觀察型沙盒、輕度策略模擬。  
**核心幻想**：玩家化身「無形的管理者」，觀察一個 5×5 微型村落中，兩位 Q 版村民如何在有限的土地與糧食資源下，自主開墾、耕作、採收，並在嚴格的日夜與土地老化循環中努力維持生存。  

遊戲強調「非直接控制」的樂趣：一切由簡單但精心設計的 AI 規則驅動，玩家主要負責觀察、加速時間、記錄有趣的模擬過程，並從中感受「土地輪作」與「隨機探索」的美麗與殘酷。

**核心樂趣迴圈**：
- 觀察村民每日外出探索 → 開墾草地或採收成熟田地 → 帶回糧食補給寨營 → 夜晚返回休息。
- 見證土地狀態隨時間自然循環（草地 → 田地 → 空地 → 草地），體會永續農業的挑戰。
- 體驗 emergent behavior：隨機移動 + 有限時間 + 體力成本，可能導致土地退化或意外繁榮。

**目標受眾**：
- 喜愛《Stardew Valley》、《The Sims》自由意志模式、《RimWorld》pawn 行為觀察的玩家。
- 喜歡短時間掛機觀察或長時間記錄「村落命運」的療癒系玩家。
- 對生態模擬、細胞自動機、agent-based modeling 有興趣的玩家與教育用途。

**獨特賣點 (USP)**：
- 緊湊世界（5×5）卻擁有完整閉環生態與經濟；外圍可走遠、夜間回營需多步。
- 完全自主 AI（無玩家微操），但規則透明且可預測，適合深度觀察。
- 土地「開墾—耗竭—休耕—恢復」四階段循環是核心敘事與機制。
- Q 版療癒畫風 + 清晰 HUD 標示晝夜；視覺輕鬆但機制有深度。

## 2. 遊戲目標與勝負條件

**主要目標（觀察型）**：
- 維持村落運作，讓初始兩位村民盡可能活滿「五年」（60 日）壽命。
- 最大化累計糧食生產量與土地有效利用率（農田比例）。
- 觀察並記錄不同隨機種子（seed）下產生的不同「村落結局」。

**失敗條件**：
- 人口歸零（村民因體力耗盡、無法返回、或壽終正寢而死亡）。

**無傳統勝利畫面**：屬於純沙盒觀察模擬。未來版本可加入「挑戰模式」（例如「維持村落存活 5 年」或「達成糧食庫存 XXX」）並給予結局評價與重播功能。

## 3. 世界與環境設計

### 3.1 地圖結構
- **嚴格 5×5 網格**（共 25 格）。
  - **中心格 (2,2)**：**寨營**（Home Base）
    - 不可耕種。
    - 功能：糧食全局儲存、村民每日開始時補給體力、夜晚強制休息處。
    - 視覺：小木屋 / 帳篷區 + 柵欄 + 明顯的糧食儲藏堆（糧袋數量隨庫存即時變化）。
  - **外圍 24 格**：**土地**（可互動耕作區）。
    - 三種狀態相互轉換。

### 3.2 土地狀態與循環（核心機制）
土地狀態機：
1. **草地 (Grassland)**：初始野生狀態。村民可移動至此執行「開墾」動作，將其轉為「田地」。
2. **田地 (Farmland)**：可生產糧食的耕作地。具有 `pending_harvest` 數值。
3. **空地 (Fallow / Empty)**：休耕或耗竭狀態。無法生產，需時間自然恢復。

**狀態轉換規則**（以「日」為單位計時）：
- 草地 → 村民開墾動作 → **立即** 變為田地（age = 0）。
- 田地存在滿 **12 日** 後 → 自動變為空地。
- 空地存在滿 **12 日** 後 → 自動變為草地。

此設計模擬真實農業「開墾 → 連續耕作耗地力 → 必須休耕讓土地恢復」的輪作概念。村民若過度開墾而 neglect 老田地，土地會快速退化，造成糧食危機。

**初始配置建議**（MVP）：
- 所有 24 塊外圍土地初始為「草地」。
- 寨營初始糧食庫存：10 單位。
- 兩位村民初始位置：寨營，體力 10/10。

### 3.3 時間系統
- **壓縮時間**：一年 = 12 日（每「日」概念上對應現實一個月）。
- **每日結構**：僅 6 小時（白天 3 小時 + 晚上 3 小時）。
- **最小刻度**：以「小時」為 tick（每 tick 推進 1 小時）。
  - 白天時段（tick 1–3）：適合外出行動、開墾、採收。
  - 晚上時段（tick 4–6）：**強制返回寨營休息**。
- **日夜資訊**：以 HUD 顯示白天／夜晚（☀/☾）；**v0.1 不做**全畫面背景或棋盤色調切換（固定淺色，避免觀看疲勞）。全畫面光線／陰影留給未來 3D 照明。
- **土地計時器**：獨立使用「累計日數」或每格記錄 `state_enter_day`，每日結束時檢查是否達到 12 日門檻。

## 4. 資源與經濟系統

### 4.1 糧食 (Food / Grain)
- **全局共享**：僅儲存在寨營（單一數字顯示）。
- **生產來源**：**每天開始（新一日 tick 1）**，每塊處於「田地」狀態的土地自動 +1 至該 tile 的 `pending_harvest`。
  - 若多日未採收，`pending_harvest` 可累積，但**每格上限為 3**（超過不再增加；鼓勵定期採收）。
- **消耗**：
  - 每日開始時，村民優先從寨營取出糧食補充體力（**1 單位糧食 = 恢復 3 點體力**）。
  - 若庫存不足，則部分補充（體力不會超過上限）。
- **運輸**：村民採收後將 `pending_harvest` 全部加入自身 `carried_food`，返回寨營後自動存入全局庫存。

### 4.2 體力 (Stamina)
- **上限**：10 點。
- **消耗**：
  - 移動一格（含對角線）：**消耗 1 點體力 + 1 小時**。
  - 開墾草地轉田地：**額外消耗 1 點體力 + 1 小時**。
  - 採收田地：**僅消耗 1 小時**（無體力消耗）。
- **恢復**：
  - **主要來源**：每日開始在寨營由糧食快速補滿（或部分補）。
  - **次要來源（建議加入以提升樂趣）**：夜晚在寨營休息時，每夜被動恢復 +2 點體力（避免純依賴糧食的脆性）。
- **死亡風險**：體力歸零且無法移動時，進入「虛弱」狀態。若晚上仍無法返回寨營，則有死亡機率（或直接死亡，依 playtest 調整）。

## 5. 村民 AI 行為設計（核心遊戲性）

初版限制：
- 僅有**成年男性 + 成年女性**各一名。
- **無繁殖、無小孩**（壽命固定為 5 年 = 60 日；1 年 = 12 日壓縮曆）。
- 兩人外觀、動作微有差異，但行為邏輯完全相同（未來可加入個性差異）。

### 5.1 行為優先順序（建議使用 Behavior Tree 或簡單 Priority List 實作）
1. **夜間強制返回（最高優先）**  
   若進入或處於晚上時段，且目前位置不在寨營 → 立即計算最短路徑（A*，8 方向移動）往寨營移動。即使體力不足也嘗試執行（增加緊張感）。

2. **每日開始補給階段**（新一日 tick 1，在寨營時觸發）  
   - 先將身上 `carried_food` 全部存入寨營庫存。
   - 消耗寨營糧食將體力補至 10（不足則盡量補）。
   - 若體力仍大於 0 且為白天時段 → 選擇**隨機一個外圍方向**，開始「探索遠征」模式。

3. **探索與工作模式**（白天外出時）  
   村民以「探索」狀態在外圍 8 格間移動（可相鄰移動，不一定要經由寨營）。  
   **到達任一土地格子後立即判斷**：
   - **草地**：執行開墾動作（花 1 小時 + 1 體力），將該格轉為「田地」，**立即設定返回旗標**。
   - **田地 且 `pending_harvest` > 0**：執行採收動作（花 1 小時），將全部 pending 加入 `carried_food`，pending 清零，**設定返回旗標**。
   - **田地（已採收，pending = 0）或空地**：**不返回**，選擇「相對遠離寨營中心」的方向或隨機相鄰外圍格子，繼續探索（避免空手立即返回寨營）。

4. **返回模式**  
   一旦設定返回旗標，或完成開墾/採收動作後 → 計算最短路徑返回寨營（相鄰格 1 步；角落到中心 Chebyshev = 2 步）。

### 5.2 移動與路徑
- 支援 **8 方向**移動（上下左右 + 對角線）。
- 每移動一格固定消耗 1 體力 + 1 小時。
- 5×5 上：與寨營相鄰的格 1 步可回營；角落需 2 步；外圍格之間可相鄰移動。
- 建議實作簡單 BFS / A* 或貪婪路徑（狀態空間仍小，效能無壓力）。

### 5.3 多村民互動
- 初版允許兩位村民停留在同一格（簡化實作）。
- 未來可加入簡單碰撞避免或「一起行動」的社交模擬。

## 6. 視覺風格與呈現

### 6.1 整體視覺
- **視角**：**固定 2.5D**（等角或 3/4 俯角），完整顯示整個 5×5 區域，像一個可旋轉或靜態的微縮立體模型。**無攝影機移動或縮放**。
- **畫風**：
  - **人物**：Q 版漫畫風（Chibi / 療癒大頭比例），線條乾淨可愛。男/女有明顯服裝與髮型區別。
  - **環境**：**淡雅清新手繪 / 水彩風**，色調柔和（草綠、土黃、米白、淡藍為主），避免高對比與寫實細節。
- **日夜呈現（v0.1）**：
  - **HUD** 常駐顯示白天／夜晚（文字 + 簡單圖示）。
  - **畫面背景與土地色塊維持固定淺色**，不隨日夜全畫面壓暗（舒適優先）。
  - **未來 3D**：再考慮方向光、陰影、營地燈光與夜間氛圍（不在 2D MVP 範圍）。

### 6.2 動畫與特效需求
- **人物動畫**（每人至少）：
  - Idle（營地休息）
  - Walk（8 方向）
  - Hoe / Till（開墾草地）
  - Harvest / Pick（採收田地）
  - Carry（帶著糧食走路）
  - Sleep（晚上在寨營）
- **環境特效**：
  - 開墾時土粒子飛濺
  - 採收時金色光點 + 收成特效
  - 田地「可採收」時的輕微發光或作物搖曳
  - 空地呈現乾燥、龜裂或少量雜草的視覺差異

### 6.3 UI / UX 設計原則
極簡、非侵入式，讓玩家專注觀察模擬本身。
- **常駐 HUD**（左上或頂部）：
  - 目前時間：第 X 年 / 第 Y 日 / 白天｜晚上 圖示
  - 寨營糧食庫存數量（大數字 + 糧袋圖示）
- **統計面板**（可收合，右側或底部）：
  - 土地統計：草地 X / 田地 X / 空地 X
  - 人口與平均存活天數
  - 累計生產糧食總量
- **互動提示**：
  - Hover 土地格：顯示目前狀態、pending_harvest、距離轉換剩餘日數
  - Hover 村民：顯示姓名/性別、目前體力條、攜帶糧食、當前行動描述（「正在開墾...」「返回寨營中」）
- **控制介面**（底部或側邊浮動）：
  - 暫停 / 播放
  - 速度滑桿（1×、2×、5×、10×、最大速度）
  - 重置模擬 / 切換 Seed 按鈕
  - 記錄當前 replay / 匯出統計按鈕（未來功能）

## 7. 技術實現建議

### 7.1 推薦引擎
- **首選**：Godot 4.x（輕量、原生 TileMap + AnimationPlayer 優秀、BehaviorTree 插件豐富、輸出體積小、適合長時間掛機）。
- **備選**：Unity 2022+（URP 2D 或 Sprite Renderer + Tilemap）。
- **快速原型**：可先用 TypeScript + PixiJS / Phaser 做出 web 版驗證核心迴圈（因為地圖極小，開發極快）。

### 7.2 資料結構建議
```gdscript
# 簡化示例（Godot GDScript 風格）
enum TileState { GRASS, FARM, EMPTY }

class Tile:
    var state: TileState
    var age_days: int = 0          # 進入當前狀態的天數
    var pending_harvest: int = 0

class Agent:
    var pos: Vector2i
    var stamina: int = 10
    var carried_food: int = 0
    var gender: String             # "male" / "female"
    var age_days: int = 0
    var state: AgentState          # RESTING, EXPLORING, HARVESTING, CONVERTING, RETURNING...
    var path: Array[Vector2i] = []

var camp_food: int = 10
var current_day: int = 0
var current_hour: int = 0          # 0~5
var tiles: Array[Tile]             # 5x5
var agents: Array[Agent]
```

### 7.3 模擬主迴圈流程
1. 每小時 tick 開始。
2. 推進時間（hour++，若 hour == 6 則 day++、hour=0、處理土地 age 增加與狀態轉換）。
3. 若為新一日開始（hour == 0）：所有田地 `pending_harvest += 1`。
4. 更新所有 Agent：
   - 檢查夜間強制返回。
   - 檢查每日開始補給與外出。
   - 執行當前行動（移動、開墾、採收）。
   - 消耗體力與時間。
5. 更新視覺（位置補間、動畫狀態、粒子；日夜以 HUD 為準，2D 不做全畫面 tint）。
6. 統計更新與 UI 刷新。

### 7.4 其他技術建議
- 使用 **seeded RNG**，讓玩家可重現有趣的模擬過程。
- 支援完整 state 序列化（JSON），可 save / load 或分享 seed。
- 實作簡單 debug 視覺化（顯示每位村民當前 path、目標 tile）。
- 音效：極簡（環境 ambient + 日夜切換的輕音樂，未來再加）。

## 8. MVP 範圍與排除項目

### 包含（v0.1 必須完成）
- 完整 5×5 網格 + 三種土地狀態 + 年齡計時器 + 每日生產機制（pending 上限 3）。
- 兩位固定村民（Q版 sprite + 基本 8 方向 walk + 開墾/採收/休息動畫）。
- 完整自主 AI 行為（第 5 節所有優先順序）。
- 日夜循環 + 土地狀態視覺切換 + 簡單粒子特效。
- 基本 UI（時間、糧食、土地統計、速度控制、暫停、重置）。
- 統計追蹤與簡單 replay 功能（記錄每日糧食與土地變化）。
- 至少 3 個預設初始 seed。

### 明確排除（v0.1 不做）
- 繁殖與小孩系統。
- 玩家直接控制村民或下達指令（純觀察）。
- 更多資源類型（水、肥料、工具耐久）。
- 天氣、災害、季節（beyond day/night）。
- 音效、配樂、語音。
- 成就、排行榜、雲端存檔、多人觀察。
- 大型地圖或滾動視角。

**預估開發人力與時間**（使用 AI 輔助 coding）：
- 1 名程式（使用 Godot + Cursor / Continue / Grok Build） + 1 名藝術（或 AI 生成 + 微調） → 2–4 週可出可玩 MVP 原型。
- 核心機制與 AI 行為可先以純文字模擬或 Python 快速驗證，再移植到引擎。

## 10. 風險控管與平衡建議

**主要風險**：
- **參數不平衡**：初始糧食 10、體力 10、生產 1/日 等數值需經過多輪 playtest 調整，避免「永遠有餘糧」或「開局 3 日就崩潰」。
- **AI 行為死鎖**：村民可能在壞土地間無效亂走而錯過夜間返回。需加入「體力過低時優先返回」或「最長探索時間上限」保險機制。
- **隨機性娛樂價值**：確保即使土地快速退化，村民仍會努力開新墾，產生「雖敗猶榮」的觀察樂趣。
- **Scope creep**：嚴格遵守 v0.1 僅做「觀察核心」，任何玩家控制或新資源都延後至 v0.2 之後。

**平衡測試重點**：
- 不同 seed 下，一壽命週期（5 年 / 60 日）內的平均存活率與最終糧食庫存分布。
- 當所有土地進入空地狀態時，村民是否仍有足夠體力與時間開墾新草地。
- 夜間返回是否過於嚴苛（可微調晚上時段長度或加入「緊急返回」機制）。
